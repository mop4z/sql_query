# sql_query

A type-safe, dynamic SQL query builder for PostgreSQL built on top of [sqlx](https://github.com/launchbadge/sqlx).

## Features

- **Type-safe columns** via derive macro `SqlCols` -- column names are enum variants, not strings
- **Full CRUD** -- SELECT, INSERT, UPDATE, DELETE builders
- **Composable expressions** -- AND/OR grouping, NOT, subqueries, CASE/WHEN
- **CTE support** -- `WITH` clauses for complex queries
- **Parameterized queries** -- all values are bound parameters, never interpolated
- **Async execution** -- `fetch_all`, `fetch_one`, `fetch_optional`, `execute` via sqlx

## Setup

Define your table struct and derive the necessary traits:

```rust
use sql_query::{SqlQ, SqlExpr, SqlCols, Table, Id, define_id};
use sqlx::FromRow;

define_id!(UserId);

#[derive(Debug, FromRow, SqlCols)]
struct Users {
    id: UserId,
    name: String,
    age: i32,
    email: Option<String>,
}

impl Table for Users {
    type Col = UsersCol;     // generated by SqlCols
    type Id = UserId;
    const TABLE_NAME: &'static str = "users";
    const PRIMARY_KEY: &'static str = "id";
}
```

`#[derive(SqlCols)]` generates a `UsersCol` enum with variants `Id`, `Name`, `Age`, `Email` that serialize to their snake_case database column names.

## Query Building

All queries start from `SqlQ` and follow the builder pattern:

```
SqlQ::select/insert/update/delete() -> builder -> .build() -> BoundQuery -> .execute(pool)
```

### SELECT

```rust
use sql_query::{SqlQ, SqlExpr, SqlOp, SqlFn, SqlOrder, SqlParam};

type UExpr = SqlExpr<Users>;

// SELECT * FROM "users"
let query = SqlQ::select::<Users>().build()?.build_as::<Users>();

// SELECT "users".name, "users".age FROM "users"
let query = SqlQ::select::<Users>()
    .from([UExpr::column(UsersCol::Name), UExpr::column(UsersCol::Age)])
    .build()?
    .build_as::<Users>();

// SELECT * FROM "users" WHERE "users".name = $1 AND "users".age > $2
let query = SqlQ::select::<Users>()
    .filter([
        UExpr::eq(UsersCol::Name, "alice"),
        UExpr::gt(UsersCol::Age, 18i32),
    ])
    .build()?
    .build_as::<Users>();

// SELECT DISTINCT "users".name FROM "users"
// WHERE 1=1 AND "users".age >= $1
// ORDER BY "users".name ASC
// LIMIT $2 OFFSET $3
let query = SqlQ::select::<Users>()
    .distinct()
    .from([UExpr::column(UsersCol::Name)])
    .filter([UExpr::gte(UsersCol::Age, 18i32)])
    .order_by(UExpr::column(UsersCol::Name), SqlOrder::Asc)
    .limit(10)
    .offset(20)
    .build()?
    .build_as::<Users>();

// Execute
let users: Vec<Users> = query.fetch_all(&pool).await?;
```

#### Column Functions & Aliases

```rust
// SELECT COUNT("users".id) AS total, AVG("users".age) AS avg_age FROM "users"
SqlQ::select::<Users>()
    .from([
        UExpr::count(UsersCol::Id).alias("total"),
        UExpr::avg(UsersCol::Age).alias("avg_age"),
    ])
```

Available functions: `Count`, `Sum`, `Avg`, `Min`, `Max`, `Lower`, `Upper`, `Length`, `Trim`, `Concat`, `Substring`, `Coalesce`, `Now`.

#### GROUP BY & HAVING

```rust
// SELECT "users".age, COUNT("users".id) AS count FROM "users"
// GROUP BY "users".age
// HAVING 1=1 AND COUNT("users".id) = $1
SqlQ::select::<Users>()
    .from([
        UExpr::column(UsersCol::Age),
        UExpr::count(UsersCol::Id).alias("count"),
    ])
    .group_by([UExpr::column(UsersCol::Age)])
    .having([
        UExpr::count(UsersCol::Id).op(SqlOp::Eq).val(5i32),
    ])
```

#### JOINs

```rust
use sql_query::SqlJoin;

// SELECT * FROM "users"
// INNER JOIN "posts" ON "posts".user_id = "users".id
SqlQ::select::<Users>()
    .join::<Posts, Users>(
        SqlJoin::Inner,
        PExpr::column(PostsCol::UserId),
        SqlOp::Eq,
        UExpr::column(UsersCol::Id),
    )
```

Join types: `Inner`, `Left`, `Right`, `FullOuter`, `Cross`.

#### Subqueries

```rust
// SELECT * FROM "users"
// WHERE "users".id IN (SELECT "posts".user_id FROM "posts" WHERE "posts".active = $1)
let sub = SqlQ::select::<Posts>()
    .from([PExpr::column(PostsCol::UserId)])
    .filter([PExpr::eq(PostsCol::Active, true)]);

SqlQ::select::<Users>()
    .filter([UExpr::in_select(UsersCol::Id, sub)])
```

### INSERT

```rust
// INSERT INTO "users" (name, age) VALUES ($1, $2)
SqlQ::insert::<Users>()
    .values([
        UExpr::eq(UsersCol::Name, "alice"),
        UExpr::eq(UsersCol::Age, 30i32),
    ])?
    .build()?
    .build()
    .execute(&pool).await?;

// Multi-row insert
SqlQ::insert::<Users>()
    .values_nested([
        vec![UExpr::eq(UsersCol::Name, "alice"), UExpr::eq(UsersCol::Age, 30i32)],
        vec![UExpr::eq(UsersCol::Name, "bob"), UExpr::eq(UsersCol::Age, 25i32)],
    ])?
    .build()?;
```

#### ON CONFLICT (Upsert)

```rust
use sql_query::SqlConflict;

// INSERT ... ON CONFLICT (name) DO UPDATE SET age = EXCLUDED.age
SqlQ::insert::<Users>()
    .values([UExpr::eq(UsersCol::Name, "alice"), UExpr::eq(UsersCol::Age, 30i32)])?
    .on_conflict(SqlConflict::DoUpdate {
        conflict_cols: vec![UsersCol::Name],
        update_cols: vec![UsersCol::Age],
    })

// INSERT ... ON CONFLICT DO NOTHING
SqlQ::insert::<Users>()
    .values([...])?
    .on_conflict(SqlConflict::DoNothing)

// INSERT ... ON CONFLICT ON CONSTRAINT users_pkey DO UPDATE SET ...
SqlQ::insert::<Users>()
    .values([...])?
    .on_conflict(SqlConflict::OnConstraint {
        name: "users_pkey",
        update_cols: vec![UsersCol::Name, UsersCol::Age],
    })
```

#### RETURNING

```rust
// INSERT ... RETURNING *
SqlQ::insert::<Users>()
    .values([...])?
    .returning_all()

// INSERT ... RETURNING "users".id, "users".name
SqlQ::insert::<Users>()
    .values([...])?
    .returning([UExpr::column(UsersCol::Id), UExpr::column(UsersCol::Name)])
```

### UPDATE

```rust
// UPDATE "users" SET "users".name = $1 WHERE 1=1 AND "users".id = $2 RETURNING *
SqlQ::update::<Users>()
    .set([UExpr::eq(UsersCol::Name, "new_name")])
    .filter([UExpr::eq(UsersCol::Id, 1i32)])
    .returning_all()
    .build()?;

// UPDATE with NOW()
SqlQ::update::<Users>()
    .set([
        UExpr::column(UsersCol::Name).op(SqlOp::Eq).val_fn(SqlFn::Now),
    ])
```

#### UPDATE ... FROM (multi-table)

```rust
// UPDATE "users" SET "users".name = $1
// FROM "posts"
// WHERE 1=1 AND "users".id = $2 AND "posts".title = $3
SqlQ::update::<Users>()
    .set([UExpr::eq(UsersCol::Name, "updated")])
    .from::<Posts>()
    .filter([UExpr::eq(UsersCol::Id, 1i32)])
    .filter([PExpr::eq(PostsCol::Title, "hello")])
```

### DELETE

```rust
// DELETE FROM "users" WHERE 1=1 AND "users".id = $1
SqlQ::delete::<Users>()
    .filter([UExpr::eq(UsersCol::Id, 1i32)])
    .build()?;

// DELETE all rows (requires explicit opt-in)
SqlQ::delete::<Users>()
    .delete_all()
    .build()?;

// DELETE ... RETURNING *
SqlQ::delete::<Users>()
    .filter([UExpr::eq(UsersCol::Id, 1i32)])
    .returning_all()
    .build()?;
```

Calling `.build()` without `.filter()` or `.delete_all()` returns an error to prevent accidental full-table deletes.

## Expression Composition

### AND / OR Grouping

```rust
// WHERE 1=1 AND ("users".name = $1 OR "users".name = $2)
.filter([
    UExpr::eq(UsersCol::Name, "alice")
        .or(UExpr::eq(UsersCol::Name, "bob")),
])

// WHERE 1=1 AND ("users".name = $1 AND "users".age > $2)
.filter([
    UExpr::eq(UsersCol::Name, "alice")
        .and(UExpr::gt(UsersCol::Age, 18i32)),
])

// Nested: WHERE 1=1 AND ("users".name = $1 OR ("users".name = $2 AND "users".age > $3))
.filter([
    UExpr::eq(UsersCol::Name, "alice")
        .or(UExpr::eq(UsersCol::Name, "bob")
            .and(UExpr::gt(UsersCol::Age, 30i32))),
])
```

### NOT

```rust
// WHERE 1=1 AND NOT ("users".name = $1)
.filter([UExpr::eq(UsersCol::Name, "alice").not()])

// NOT with OR: AND NOT (("users".name = $1 OR "users".name = $2))
.filter([
    UExpr::eq(UsersCol::Name, "alice")
        .or(UExpr::eq(UsersCol::Name, "bob"))
        .not(),
])
```

### CASE / WHEN / THEN / ELSE

```rust
// CASE WHEN "users".age = $1 THEN $2 ELSE $3 END AS age_group
UExpr::eq(UsersCol::Age, 18i32)
    .then(UExpr::empty().val(SqlParam::String("minor".into())))
    .else_(UExpr::empty().val(SqlParam::String("adult".into())))
    .alias("age_group")
```

Both `.then()` and `.else_()` are required -- using one without the other returns `CaseRequiresThenAndElse`.

### BETWEEN

```rust
// WHERE 1=1 AND "users".age BETWEEN $1 AND $2
.filter([UExpr::between(UsersCol::Age, 18i32, 65i32)])
```

### EXISTS / NOT EXISTS

```rust
// WHERE 1=1 AND EXISTS (SELECT * FROM "posts" WHERE ...)
let sub = SqlQ::select::<Posts>().filter([PExpr::eq(PostsCol::Active, true)]);
.filter([UExpr::exists(sub)])

// NOT EXISTS
.filter([UExpr::not_exists(sub)])
```

### LIKE / ILIKE

```rust
// WHERE 1=1 AND "users".name LIKE $1
.filter([UExpr::like(UsersCol::Name, "%alice%")])

// Case-insensitive
.filter([UExpr::ilike(UsersCol::Name, "%alice%")])
```

### IS NULL / IS NOT NULL

```rust
.filter([UExpr::is_null(UsersCol::Email)])
.filter([UExpr::is_not_null(UsersCol::Email)])
```

### IN / NOT IN

```rust
// WHERE 1=1 AND "users".name = ANY ($1)
.filter([UExpr::any(UsersCol::Name, vec!["alice".to_string(), "bob".to_string()])])

// With subquery
.filter([UExpr::in_select(UsersCol::Id, sub)])
.filter([UExpr::not_in_select(UsersCol::Id, sub)])
```

### JSON Operators

```rust
// "users".data -> 'key'
UExpr::json_get(UsersCol::Data, "key")

// "users".data ->> 'key'
UExpr::json_get_text(UsersCol::Data, "key")

// Verbose form for #> and #>>
UExpr::column(UsersCol::Data).op(SqlOp::JsonPath).val("{a,b}")
UExpr::column(UsersCol::Data).op(SqlOp::JsonPathText).val("{a,b}")
```

## CTEs (Common Table Expressions)

```rust
// WITH active AS (SELECT * FROM "users" WHERE ...) SELECT * FROM "users" WHERE ...
SqlQ::with([
    ("active", SqlQ::select::<Users>().filter([UExpr::eq(UsersCol::Age, 18i32)])),
])
.select::<Users>()
.filter([UExpr::eq(UsersCol::Name, "alice")])
```

CTEs work with all statement types: `.select()`, `.delete()`, `.insert()`, `.update()`.

Multiple CTEs:

```rust
SqlQ::with([
    ("young", SqlQ::select::<Users>().filter([UExpr::eq(UsersCol::Age, 18i32)])),
    ("old", SqlQ::select::<Users>().filter([UExpr::eq(UsersCol::Age, 65i32)])),
])
.select::<Users>()
```

## Execution

Every query builder produces an `UnbindedQuery` via `.build()`, which is then finalized:

```rust
// For SELECT queries returning typed rows
let users: Vec<Users> = SqlQ::select::<Users>()
    .filter([...])
    .build()?
    .build_as::<Users>()
    .fetch_all(&pool).await?;

// For single row
let user: Users = query.build()?.build_as::<Users>().fetch_one(&pool).await?;

// For optional row
let user: Option<Users> = query.build()?.build_as::<Users>().fetch_optional(&pool).await?;

// For scalar values (e.g., COUNT)
let count: i64 = query.build()?.build_scalar::<i64>().fetch_one(&pool).await?;

// For INSERT/UPDATE/DELETE (rows affected)
let result = query.build()?.build().execute(&pool).await?;
```

## Custom Postgres Enums

Use `#[derive(SqlParamEnum)]` to make your Postgres enum types usable as bind values:

```rust
use sql_query::SqlParamEnum;

#[derive(Clone, Debug, sqlx::Type, SqlParamEnum)]
#[sqlx(type_name = "currency_type", rename_all = "snake_case")]
pub enum CurrencyType {
    Fiat,
    Crypto,
}

// Use directly in expressions -- the Postgres enum OID is preserved
SqlExpr::eq(CurrencyCol::CurrencyType, CurrencyType::Fiat)
```

## Error Handling

`SqlQueryError` is returned for invalid expression combinations:

| Error | Cause |
|-------|-------|
| `AndOrBothSet` | `.and()` and `.or()` on same expression |
| `BetweenMissingBounds` | `BETWEEN` without both bounds |
| `ExistsMissingSelect` | `EXISTS` without `.select()` |
| `InsertValuesAlreadySet` | `.values()` called twice |
| `DeleteRequiresFilterOrDeleteAll` | DELETE without `.filter()` or `.delete_all()` |
| `CaseRequiresThenAndElse` | `.then()` without `.else_()` or vice versa |
